## 🔗 框架能力查询

**在编写后端 API 或 Service 层代码时**,请确认以下情况:

### 必须先查阅 `后端脚手架.md` 的场景

- ✅ 需要**统一返回格式**(⚠️ 框架已自动封装,Controller 直接返回业务对象)
- ✅ 需要**分页查询**(PageModel、IPage 使用)
- ✅ 需要**全局异常处理**(ServiceException 抛出,⚠️ 框架自动处理)
- ✅ 需要**请求参数校验**(Validation 注解)
- ✅ 需要**对象转换**(Entity ↔ BO ↔ VO)
- ✅ 需要**数据库操作**(MyBatis-Plus BaseMapper/IService)

### 无需查阅的场景

- ❌ 定义 Request/Response 对象结构
- ❌ 编写纯业务逻辑(已明确使用的框架能力)

## API 设计规范

### 请求参数（Request）

- **接口参数数量 ≤ 3 个**：可直接传递独立参数
- **接口参数数量 > 3 个**：优先使用 POST 方式，并将参数封装为请求对象
- **命名规范**：`模块名 + 接口方法名 + Request` 后缀

### 响应结果（Response）

- **命名规范**：`模块名 + 接口方法名 + Response` 后缀

---

## 接口类

- **标准命名**：以 `I` 开头，如 `IEnterpriseService`
- **例外情况**：
  1. 若模块首字母已是 `I`，无需额外添加前缀（如 `IndustrySearchService` 而非 `IIndustrySearchService`）
  2. 常量接口类可不使用 `I` 前缀

## 接口实现类

- **命名规范**：以 `Impl` 结尾

---

## 数据传输规范

### Controller → Service

#### 新增/编辑操作

| 场景 | 传输方式 |
|------|----------|
| 单实体操作 | 封装为实体对象（Entity//Document） |
| 多实体操作 | 封装为 Condition 对象 |

#### 查询/删除操作

| 参数数量 | 传输方式 |
|----------|----------|
| ≤ 3 个参数 | 直接传递独立参数 |
| > 3 个参数 | 封装为 Condition 对象 |

---

### Service → Service/Dao

#### 新增操作

- 传递实体对象（Entity/Document）

#### 编辑操作

| 场景 | 传输方式 |
|------|----------|
| 全字段更新 | 传递实体对象（Entity/Document） |
| 部分字段更新 ≤ 3 个 | 直接传递独立参数 |
| 部分字段更新 > 3 个 | 封装为 Condition 对象 |

#### 查询/删除操作

- 根据参数数量参照 Controller → Service 规范

---

### Dao → Service

| 数据类型 | 返回对象                |
|----------|---------------------|
| 正常数据源（单条/列表） | 实体                  |
| 聚合数据/连表查询 | BO（Business Object） |

---

### Service → Controller

- **统一使用 BO（Business Object）** 进行数据返回

---

## 分层对象说明

| 对象类型            | 全称                   | 用途 |
|-----------------|----------------------|------|
| Entity/Document | 实体                   | 数据库表映射对象，用于持久层 |
| BO              | Business Object      | 业务对象，用于业务逻辑层 |
| Condition       | Query Condition      | 查询条件封装对象 |

## Maven 构建优化

### 多线程构建规范

**所有 Maven 命令必须使用 `-T` 参数启用多线程构建**

**线程数计算**: `max(CPU核心数, 3)`

**注意**:
1. 使用 `-T 数字`，不要使用 `-T 1C` 或 `-T 2C`
2. 最低保证 3 个线程
3. 不要跳过单元测试（除非明确要求）
