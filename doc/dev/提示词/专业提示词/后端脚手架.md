# fun-framework-java 脚手架能力索引

**用途**: 编写新代码涉及通用功能时查阅，避免重复实现框架已有能力。

**仓库**: GitHub/Gitee `fanzezhen/fun-framework-java`  
**版本**: 动态 (Spring Boot 4.x+, Java 21+), 查看 pom.xml  
**开发流程**: 查本文档 → 找模块 → 读 README → 看测试 → 写代码

**核心原则**:
1. 先查后写
2. 依赖版本由框架父 POM 统一管理，禁止覆盖
3. 不确定时查文档或询问用户

## 框架模块总览

### 1. 核心基础模块 (fun-framework-core)

#### 1.1 核心模型 (fun-framework-core-model)
**功能**: 通用数据模型和工具  
**文档**: [README.md](https://github.com/fanzezhen/fun-framework-java/blob/master/fun-framework-core/fun-framework-core-model/README.md) ([Gitee](https://gitee.com/fanzezhen/fun-framework-java/blob/master/fun-framework-core/fun-framework-core-model/README.md))

**核心内容**:
- `Result<T>`: 统一返回结果 (code/message/data)
- `PageResult<T>`: 分页模型
- `MapperFacadeUtil`: 对象映射工具 (基于 Orika)
- `BaseBO/BaseDTO`: 业务对象基类 (包含 id/createTime/createUserId)
- `BaseTenantBO/BaseTenantDTO`: 多租户业务对象基类
- `IGenericEntity<P>`: 非 MyBatis Entity 接口

**继承规范**:
- BO → 继承 `BaseBO<P>` (Service 层返回)
- DTO → 继承 `BaseDTO<P>` (Controller 层入参/出参)
- 多租户 → 使用 `BaseTenant*`
- 非 MyBatis Entity → 实现 `IGenericEntity<P>`

**MapperFacadeUtil 用法**:
```java
// 单个对象转换
UserBO bo = MapperFacadeUtil.map(entity, UserBO.class);

// 集合转换
List<UserBO> list = MapperFacadeUtil.mapAsList(entityList, UserBO.class);

// 分页转换
PageResult<UserBO> boPage = MapperFacadeUtil.page(entityPage, UserEntity.class, UserBO.class);
```

**注意**: 
- Controller 返回值自动封装为 `Result<T>`, 无需手动调用 `Result.success()`
- Spring Boot 项目需添加 `fun-framework-core-springboot` 依赖

### 2. 数据持久化模块 (fun-framework-data)

#### 2.1 MyBatis-Plus 增强 (fun-framework-data-mp-starter)
- **错误码**: 120**
- **功能**: MyBatis-Plus 增强配置
- **文档**: [README.md](https://github.com/fanzezhen/fun-framework-java/blob/master/fun-framework-data/fun-framework-data-mp/fun-framework-data-mp-starter/README.md) ([Gitee](https://gitee.com/fanzezhen/fun-framework-java/blob/master/fun-framework-data/fun-framework-data-mp/fun-framework-data-mp-starter/README.md))
- **常用内容**:
  - 自动填充配置
  - 逻辑删除支持
  - 多租户支持
  - 分页插件配置
  - **BaseEntity 基类** (路径: `com.github.fanzezhen.fun.framework.mp.base.entity`)

**使用场景**:
- 数据库 CRUD 操作
- 自动填充创建时间/更新时间
- 逻辑删除
- 多租户数据隔离

**MyBatis-Plus Entity 继承规范**:

MyBatis-Plus 实体类必须继承 `com.github.fanzezhen.fun.framework.mp.base.entity` 下的 BaseEntity,根据主键策略选择:

| 主键策略 | 路径 | 主键类型 | 说明 |
|---------|------|---------|------|
| 自增 | `increment.BaseEntity` | `Integer` | 单体应用推荐 |
| UUID | `uuid.BaseEntity` | `String` | 分布式系统 |
| 雪花算法 | `snowflake.BaseEntity` | `Long` | 高并发分布式推荐 |
| 多租户 | `*.tenant.BaseTenantEntity` | 根据策略 | 额外包含 tenantId |
| 自定义主键类型 | `*.BaseGenericEntity<P>` | 泛型 `P` | 需要自定义主键类型时使用 |

BaseEntity 提供的字段(自动填充):
- `id`: 主键
- `createTime`: 创建时间 (INSERT 自动填充)
- `createUserId`: 创建人ID (INSERT 自动填充)
- `tenantId`: 租户ID (仅多租户 BaseEntity)

示例:
```java
// 自增主键
import com.github.fanzezhen.fun.framework.mp.base.entity.increment.BaseEntity;
@TableName("sys_user")
public class UserEntity extends BaseEntity {
    private String username;
}

// 多租户 + 雪花算法
import com.github.fanzezhen.fun.framework.mp.base.entity.snowflake.tenant.BaseTenantEntity;
@TableName("tenant_order")
public class OrderEntity extends BaseTenantEntity {
    private String orderNo;
}
```

## 常见错误

### 错误 1: 手动封装返回格式

```java
// 错误
@GetMapping("/user/{id}")
public Result<UserBO> getUser(@PathVariable Long id) {
    return Result.success(userService.getById(id));  // 多余
}

// 正确
@GetMapping("/user/{id}")
public UserBO getUser(@PathVariable Long id) {
    return userService.getById(id);  // 框架自动封装为 Result<T>
}
```

### 错误 2: 不使用框架异常处理

```java
// 错误
@GetMapping("/user")
public Map<String, Object> getUser() {
    try {
        // ...
    } catch (Exception e) {
        // 手动处理异常
    }
}

// 正确
@GetMapping("/user/{id}")
public UserBO getUser(@PathVariable Long id) {
    UserBO user = userService.getById(id);
    if (user == null) {
        throw new ServiceException("用户不存在");  // 框架统一处理
    }
    return user;
}
```

### 错误 3: 覆盖框架依赖版本

```xml
<!-- 错误 -->
<properties>
    <mybatis-plus.version>3.5.10</mybatis-plus.version>
</properties>

<!-- 正确：继承框架父 POM，不覆盖版本 -->
<parent>
    <groupId>cn.fanzezhen</groupId>
    <artifactId>fun-framework-parent</artifactId>
    <version>4.0.5</version>
</parent>
```

### 错误 4: 手动编写对象映射

```java
// 错误
UserBO bo = new UserBO();
bo.setId(entity.getId());
bo.setName(entity.getUserName());
// ...

// 正确：使用 MapperFacadeUtil
UserBO bo = MapperFacadeUtil.map(entity, UserBO.class);
PageResult<UserBO> boPage = MapperFacadeUtil.page(entityPage, UserEntity.class, UserBO.class);
```

### 错误示例 5: 不继承框架 BaseEntity

```java
// 错误：MyBatis-Plus Entity 不继承 BaseEntity
@TableName("sys_user")
public class UserEntity {
    @TableId(type = IdType.AUTO)
    private Integer id;
    private LocalDateTime createTime;
    private Integer createUserId;
    private String username;
}

// 正确：继承 BaseEntity
import com.github.fanzezhen.fun.framework.mp.base.entity.increment.BaseEntity;
@TableName("sys_user")
public class UserEntity extends BaseEntity {
    private String username;
    // id/createTime/createUserId 自动继承并自动填充
}
```

### 错误示例 6: BO/DTO 不继承 Base 类

```java
// 错误：BO 不继承 BaseBO
public class UserBO {
    private Integer id;
    private LocalDateTime createTime;
    private String username;
}

// 正确：继承 BaseBO
import com.github.fanzezhen.fun.framework.core.model.bo.BaseBO;
public class UserBO extends BaseBO<Integer> {
    private String username;
    // id/createTime/createUserId 自动继承,与 Entity 字段对齐
}
```

## 使用建议

**何时查阅**:
- 实现通用功能 (统一返回、分页、缓存、日志等) → 必读
- 集成第三方服务 (Redis、ES、Sa-Token 等) → 必读
- 不确定功能是否需要自己实现 → 必读
- 纯业务逻辑、前端开发 → 无需读取

**查阅流程**: 快速查询表/模块总览 → 找模块 → 读 README → 看测试 → 写代码

**选择 Base 类**:
- MyBatis-Plus Entity: 单体用 `increment.BaseEntity`, 分布式用 `snowflake.BaseEntity`, 多租户用 `*.tenant.BaseTenantEntity`
- 非 MyBatis Entity: 实现 `IGenericEntity<P>`
- BO/DTO: Service 用 `BaseBO<P>`, Controller 用 `BaseDTO<P>`, 多租户用 `BaseTenant*`
